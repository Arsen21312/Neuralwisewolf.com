JWT Decoder: анализ JSON Web Tokens

JWT (JSON Web Token) — это открытый стандарт для создания токенов доступа, которые передают информацию между сторонами в виде JSON объекта. Наш инструмент помогает анализировать и проверять JWT токены.

🎯 Структура JWT токена

JWT состоит из трех частей, разделенных точками:

1. Header (Заголовок)

{
  "alg": "HS256",  // Алгоритм подписи
  "typ": "JWT"     // Тип токена
}

2. Payload (Полезная нагрузка)

{
  "sub": "1234567890",      // Subject (субъект)
  "name": "John Doe",       // Произвольные данные
  "iat": 1516239022,        // Issued At (время выдачи)
  "exp": 1516242622         // Expiration (время истечения)
}

3. Signature (Подпись)

Подпись создается путем кодирования header и payload с использованием secret ключа и указанного алгоритма.

🔐 Стандартные claims (поля)

  • iss (Issuer) — издатель токена
  • sub (Subject) — субъект токена
  • aud (Audience) — аудитория токена
  • exp (Expiration) — время истечения
  • nbf (Not Before) — не действителен до
  • iat (Issued At) — время выдачи
  • jti (JWT ID) — уникальный идентификатор

❓ Частые вопросы

Безопасно ли декодировать JWT?
Да, header и payload只是 base64 кодированы, а не зашифрованы. Signature требует secret ключ для проверки.

Как проверить подпись JWT?
Для проверки подписи нужен secret ключ или public key. Наш инструмент показывает подпись, но не проверяет ее без ключа.

Что делать если токен истек?
Истекший токен недействителен. Нужно получить новый токен через процесс аутентификации.

Какие алгоритмы поддерживает JWT?
HS256, RS256, ES256, PS256 и другие. HS256 (HMAC) использует один ключ, RS256 (RSA) использует пару ключей.

Можно ли изменить payload после создания токена?
Изменение payload приведет к невалидной подписи. Получатель сможет обнаружить изменение.
